# Top1200 Summary

## Before

- top1000 and competitive foundations existed
- PASS1 could target a frozen band in dry-run, but reporting still read the wrong SQLite path
- a full recompute of the top1200 band degraded `172` previously locked games to `review`
- official reported progress was `1196 / 1200`

## After

- `extension200` and `top1200` catalogs are generated and measurable
- PASS1 accepts `--selection-band`
- reporting reads the same SQLite target as the runtime/admin stack
- stable lifecycle logic keeps fully valid locked games locked after safe content refreshes
- official progress is now `1200 / 1200 complete-or-better`
- local and Supabase curation states are in sync with no publish delta

## Commands Added

- `npm run enrichment:freeze-extension200`
- `npm run enrichment:freeze-top1200-band`
- `npm run enrichment:report-top1200`
